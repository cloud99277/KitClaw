# Whiteboard Memory 提取模板

> **所属 Skill**：`memory-manager`
> **用途**：指导从对话、文档中提取 Decisions / Actions / Learnings 写入 L2 Whiteboard

---

## 一、三类条目定义

### Decision（决策）

> **定义**：在两个以上方案之间做了明确的选择，且这个选择对未来有指导意义。

**判断标准**：
- ✅ 有明确的「选了 A，放弃了 B」
- ✅ 理由值得后续复用（不是随机选的）
- ❌ 只是描述现状，没有选择行为
- ❌ 临时性的、当场就撤销的决定

**好的 Decision 示例**：
```
决定用 JSON + grep 而非 chromadb 做记忆检索。
理由：零依赖约束，grep 覆盖 80% 场景，不值得引入向量数据库。
```

**不好的 Decision 示例**（不应写入）：
```
今天打算用 Python 写这个脚本。（太随意，没有对比选择）
```

---

### Action（行动）

> **定义**：承诺要做、但尚未完成的任务。是跨会话的 TODO。

**判断标准**：
- ✅ 有明确的行动主体和行动内容
- ✅ 尚未完成，需要在未来某个会话中跟进
- ❌ 已经完成的事情（记录到 Learning 里）
- ❌ 泛泛的计划，没有具体行动

**好的 Action 示例**：
```
[action] 给 memory-update.py 补充 --delete-project 参数。
触发条件：T6 测试夹具需要清理测试数据时。
```

**不好的 Action 示例**（不应写入）：
```
以后要多写单元测试。（太泛，无法跟进）
```

---

### Learning（学习）

> **定义**：从实施过程中发现的规律、反模式或踩过的坑。对未来有警示或复用价值。

**判断标准**：
- ✅ 来自真实经验（踩坑、验证、实验）
- ✅ 可以指导未来类似场景的行为
- ❌ 纯理论知识（应该放到 L3 文档里）
- ❌ 已经在文档中记录的内容（避免重复）

**好的 Learning 示例**：
```
静态分析工具的误报控制：先限范围（仅扫 scripts/），再加上下文匹配，
最后才给白名单。跳到第三层会让规则失去意义。
来源：Phase 2 CONS-001 误报事件。
```

**不好的 Learning 示例**（不应写入）：
```
Python 的 json 模块可以解析 JSON。（众所周知，无价值）
```

## 二、提取流程

```
读取对话/文档
  ↓
扫描每个段落，问：
  "这里有 Decision 吗？" → 有 → 提取并分类
  "这里有 Action 吗？"   → 有 → 提取并分类
  "这里有 Learning 吗？" → 有 → 提取并分类
  ↓
过滤重复（与现有 whiteboard 内容比较）
  ↓
格式化为标准条目
  ↓
调用 memory-update.py 写入
```

## 三、提取示例

### 原始对话片段

```
用户：我们要不要用 chromadb 做语义检索？
Agent：考虑到零依赖约束，JSON + grep 覆盖 80% 场景，不值得引入向量数据库。
      建议先用 grep，等真实需求出现再加。
用户：好的，那就用 grep。另外记得后面要加测试。
Agent：好，T6 会加测试夹具。
```

### 提取结果

**Decision**：
```json
{
  "type": "decision",
  "content": "使用 JSON + grep 做记忆检索，不引入 chromadb。理由：零依赖约束，grep 覆盖 80% 场景。",
  "project": "agent-os",
  "tags": ["architecture", "memory", "no-dependency"]
}
```

**Action**：
```json
{
  "type": "action",
  "content": "在 T6 验收中加测试夹具（写入→检索→清理）。",
  "project": "agent-os",
  "tags": ["memory-manager", "testing"]
}
```

## 四、Agent 提取指令模板

当用户说「提取这段对话中的记忆」时，使用以下提示：

```
请从以下对话/文档中提取 Whiteboard Memory 条目：

[粘贴内容]

提取规则：
1. Decision：有明确的方案选择 + 理由
2. Action：承诺要做但未完成的任务
3. Learning：从实践中发现的规律/反模式

每条条目格式：
- 类型：decision / action / learning
- 内容：一句话清晰描述（不超过 100 字）
- 项目：所属项目名称
- 标签：2-4 个关键词

过滤掉：纯理论知识、已完成的事情、过于泛泛的计划。
```

## 五、质量检查清单

写入前检查每条条目：

| 检查项 | 通过标准 |
|--------|---------|
| 类型正确 | 是 decision / action / learning 之一 |
| 内容简洁 | ≤ 100 字，一句话说清楚 |
| 有具体价值 | 不是众所周知的知识 |
| 不重复 | 与已有条目不高度相似 |
| 有项目标签 | `project` 字段不为空 |
